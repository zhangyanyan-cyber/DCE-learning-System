什么是离散选择实验？

DCE 基于随机效用理论，通过让受访者在假想选项间反复权衡，量化属性的相对重要性与支付意愿（WTP）。

🎯

属性 (Attribute)

描述选项特征的维度，每个 DCE 通常含 4–8 个属性。

📏

水平 (Level)

属性的具体取值，每个属性通常设 2–5 个水平。

📋

选择集 (Choice Set)

受访者单次面对的选项组合，通常含 2–3 个备选方案。

🔧

实验设计 (Design)

决定属性水平如何组合成选择集的方案。

💰

支付意愿 (WTP)

受访者为某属性水平变化愿意支付的最大金额。

📈

随机效用模型

U = V + ε，不同误差假设对应不同模型形式。

⚡ DCE 研究流程（点击节点查看说明）

确定研究问题→ 选择属性与水平→ 实验设计→ 问卷构建→ 数据收集→ 计量建模→ 结果解释
点击上方任一步骤查看详细说明与常见注意事项。

📌 效用函数

Vnij = β₁X₁ + β₂X₂ + ... + βₖXₖ
P(i|C) = exp(Vᵢ) / Σⱼ exp(Vⱼ)

β 系数通过最大似然估计得到，代表属性对效用的边际贡献。

📌 WTP 计算

WTP = −(βattr / βcost)
置信区间：Krinsky-Robb 法

价格系数符号应为负，WTP 正值表示愿意支付溢价。

🔬 进阶 三大模型简述

条件 Logit (CL) — 固定参数，满足 IIA，最简单；混合 Logit (MXL) — 随机参数，允许连续偏好异质性，最常用；潜在类别 (LCM) — 离散分群，适合市场细分分析。详见「模型对比」页面。

问卷设计 R 代码

三种设计方法的完整代码，使用「研究设计工具→属性生成器」可根据你的属性自动定制代码。

01 正交设计 (Orthogonal Design)

# 正交设计 — 以医疗卫生场景为例（3属性）
library(DoE.base); library(support.CEs)

attrs <- list(
  treatment_effect = c("50%","70%","90%"),
  side_effect      = c("无","轻度","中度"),
  cost             = c("200元","500元","1000元","2000元")
)
lvls <- sapply(attrs, length)  # c(3,3,4)

# 生成正交主效应阵
orth <- oa.design(nlevels = lvls, randomize = FALSE)
for(i in seq_along(attrs)) levels(orth[[i]]) <- attrs[[i]]
names(orth) <- names(attrs)

cat("正交设计（", nrow(orth), "行）:\n"); print(orth)
# 相关性检验（应接近0）
print(round(cor(data.matrix(orth)), 3))

# 生成选择集（每组2个备选）
cs <- rotation.design(attribute.names=attrs, nalternatives=2,
  nblocks=1, row.renames=FALSE, randomize=TRUE, seed=42)
print(cs)
write.csv(orth, "orthogonal_design.csv", row.names=FALSE)

02 D-optimal 设计 (idefix)

library(idefix)

attrs <- list(
  treatment_effect = c("50%","70%","90%"),
  side_effect      = c("无","轻度","中度"),
  cost             = c("200元","500元","1000元","2000元"),
  waiting_time     = c("1天","1周","1个月")
)
lvls   <- sapply(attrs, length)   # c(3,3,4,3)
coding <- rep("E", length(lvls))  # 效应编码

n_params <- sum(lvls - 1)         # 模型参数数
cand_set <- Profiles(lvls=lvls, coding=coding)

# ── 局部 D-optimal（零先验）──
set.seed(2024)
res <- CEA(lvls=lvls, coding=coding,
  par.draws = rep(0, n_params),   # 零先验 → 不依赖任何估计
  n.sets=12, n.alts=2,
  cand.set=cand_set, n.start=12, best=TRUE)
cat("D-error:", round(res$error, 6), "\n")

design <- Decode(res$design, lvls, coding, attrs)
print(design$design)

# ── 贝叶斯 D-optimal（推荐，有先验时使用）──
library(MASS)
prior_mu  <- rep(0, n_params)
prior_cov <- diag(0.5, n_params)
draws     <- mvrnorm(500, prior_mu, prior_cov)  # 正式分析用1000+

set.seed(2024)
bres <- BCEA(lvls=lvls, coding=coding,
  par.draws=draws, n.sets=12, n.alts=2,
  cand.set=cand_set, n.start=6)
cat("贝叶斯 D-error:", round(bres$error, 6), "\n")

write.csv(Decode(bres$design,lvls,coding,attrs)$design,
  "doptimal_design.csv", row.names=FALSE)

03 最大化设计 (Federov Exchange)

library(AlgDesign)

# 候选集（全因子空间）
candidates <- expand.grid(
  treatment_effect = c(50,70,90),
  side_effect      = c(1,2,3),
  cost             = c(200,500,1000,2000),
  waiting_time     = c(1,7,30)
)

frml   <- ~ treatment_effect + side_effect + cost + waiting_time
mm     <- model.matrix(frml, candidates)
n_par  <- ncol(mm) - 1
n_rows <- n_par * 4     # 设计行数 = 参数数 × 4

set.seed(2024)
opt <- optFederov(frml, data=candidates, nTrials=n_rows,
  criterion="D", nRepeats=20, nullify=1)

cat("D效率:", round(opt$Dea, 4), "\n")
print(head(opt$design, 6))

# 标签化 & 配对选择集
d <- opt$design
d$treatment_effect <- paste0(d$treatment_effect,"%")
d$side_effect <- factor(d$side_effect,1:3,c("无","轻度","中度"))
n_cs  <- floor(nrow(d)/2)
d$choice_set  <- rep(1:n_cs, each=2)[1:nrow(d)]
d$alternative <- rep(c("方案A","方案B"), n_cs)[1:nrow(d)]
write.csv(d, "maximal_design.csv", row.names=FALSE)

# ── 读取数据（替换为你的文件路径）──
# raw <- read.csv("your_data.csv")
# 以下为模拟数据示例
set.seed(42); n_resp=200; n_tasks=12
raw <- expand.grid(respondent_id=1:n_resp, task_id=1:n_tasks) %>%
  mutate(
    cost_A=sample(c(200,500,1000,2000),n(),T),
    effect_A=sample(c(50,70,90),n(),T),
    side_A=sample(1:3,n(),T),
    cost_B=sample(c(200,500,1000,2000),n(),T),
    effect_B=sample(c(50,70,90),n(),T),
    side_B=sample(1:3,n(),T),
    choice=sample(1:2,n(),T)
  )

# ── 宽→长格式转换 ──
long <- raw %>%
  pivot_longer(matches("_(A|B)$"),
    names_to=c(".value","alternative"), names_sep="_(?=[AB]$)") %>%
  mutate(
    chosen = as.integer((alternative=="A"&choice==1)|(alternative=="B"&choice==2)),
    side   = factor(side,1:3,c("无","轻度","中度")),
    respondent_id = factor(respondent_id)
  ) %>% arrange(respondent_id, task_id, alternative)

# ── 数据质量检验 ──
chk <- long %>% group_by(respondent_id,task_id) %>%
  summarise(n=sum(chosen),.groups="drop")
cat("每任务被选数分布（应全为1）:\n"); print(table(chk$n))
cat("缺失值:\n"); print(colSums(is.na(long)))

# ── 转为 mlogit 格式 ──
md <- mlogit.data(long, choice="chosen", shape="long",
  alt.var="alternative", id.var="respondent_id", chid.var="task_id")
save(long, md, file="dce_data.RData")
cat("数据已保存（n=", n_resp, "人，", n_tasks, "任务/人）\n")

library(gmnl); load("dce_data.RData")

# ── Step 1: BIC 选择最优类别数 ──
bic_df <- data.frame()
for(k in 2:5){
  m <- tryCatch(gmnl(chosen~cost+effect+side|0|0|0|1, data=md,
    model="lc", Q=k, panel=TRUE, method="bhhh",
    iterlim=500, print.level=0), error=function(e) NULL)
  if(!is.null(m)){
    n_par <- length(coef(m)); n_obs <- length(unique(md$id))
    bic_df <- rbind(bic_df, data.frame(k=k, LL=as.numeric(logLik(m)),
      AIC=-2*logLik(m)+2*n_par, BIC=-2*logLik(m)+log(n_obs)*n_par))
  }
}
print(round(bic_df,2)); best_k <- bic_df$k[which.min(bic_df$BIC)]
cat("最优类别数:", best_k,"\n")

# ── Step 2: 估计最优模型 ──
set.seed(42)
lcm <- gmnl(chosen~cost+effect+side|0|0|0|1, data=md,
  model="lc", Q=best_k, panel=TRUE, method="bhhh", iterlim=500)
summary(lcm)

# ── Step 3: 类别规模 & WTP ──
cp <- colMeans(lcm$prob.class)
cat("各类别占比:\n"); print(round(cp,3))

# 各类别WTP
bc <- matrix(coef(lcm)[!grepl("^class",names(coef(lcm)))], ncol=best_k)
rownames(bc) <- c("cost","effect","side轻度","side中度")
wtp <- apply(bc,2, function(b) -b/b["cost"])
cat("各类别WTP（元/月）:\n"); print(round(wtp[-1,],2))

# ── 点估计 ──
wtp_pt <- c(
  `效果+1%`   = -b["effect"]/bc,
  `副作用:轻度` = -b["side轻度"]/bc,
  `副作用:中度` = -b["side中度"]/bc
)
cat("WTP 点估计（元/月）:\n"); print(round(wtp_pt,2))

# ── Krinsky-Robb 置信区间（10000次参数Bootstrap）──
set.seed(2024)
draws <- mvrnorm(10000, coef(cl), vcov(cl))
wtp_d <- data.frame(
  effect = -draws[,"effect"] / draws[,"cost"],
  side_l = -draws[,"side轻度"]  / draws[,"cost"],
  side_m = -draws[,"side中度"]  / draws[,"cost"]
)
# 截尾去极端值（Poe et al.）
for(v in names(wtp_d))
  wtp_d[[v]] <- ifelse(between(wtp_d[[v]],
    quantile(wtp_d[[v]],.005), quantile(wtp_d[[v]],.995)), wtp_d[[v]], NA)

ci_tab <- data.frame(
  attribute = c("效果提升1%","轻度副作用 vs 无","中度副作用 vs 无"),
  mean  = colMeans(wtp_d, na.rm=TRUE),
  lower = apply(wtp_d, 2, quantile, .025, na.rm=TRUE),
  upper = apply(wtp_d, 2, quantile, .975, na.rm=TRUE)
)
cat("\nWTP 95% CI（元/月）:\n")
print(ci_tab %>% mutate(label=sprintf("%.1f (%.1f, %.1f)", mean,lower,upper)))

# ── 森林图 ──
ggplot(ci_tab, aes(mean, reorder(attribute,mean))) +
  geom_vline(xintercept=0, linetype="dashed", color="gray55") +
  geom_errorbarh(aes(xmin=lower,xmax=upper), height=.22,
    color="#1a6b5a", linewidth=1.1) +

  scale_x_continuous(expand=expansion(mult=c(.05,.3))) +
  theme_minimal(base_size=13) +
  labs(title="支付意愿估计（Krinsky-Robb，95% CI）",
    x="WTP（元/月）", y="")

IIA 假设（无关选项独立性）是条件 Logit 的核心约束，若违反则应改用 MXL 或 NL 模型。
library(mlogit); load("dce_data.RData")

# ── Hausman-McFadden IIA 检验 ──
# 前提：设计中至少有3个备选方案
# 完整模型（含所有备选）
full_model <- mlogit(chosen ~ cost + effect + side | 0, data=md)

# 限制模型（去掉一个备选方案）
# 注：需要设计中有第3个选项 "C"（如"都不选"）
md_restricted <- md[md$alt != "C", ]
rest_model <- mlogit(chosen ~ cost + effect + side | 0,
  data=md_restricted)

# HMF 检验（p > 0.05 → 不拒绝 IIA，CL 合理）
htest <- hmftest(full_model, rest_model)
cat("Hausman-McFadden IIA 检验:\n"); print(htest)
cat("p值:", round(htest$p.value, 4), "\n")
cat(ifelse(htest$p.value > 0.05,
  "✓ 不拒绝 IIA（p > 0.05），条件Logit 假设合理",
  "✗ 拒绝 IIA（p < 0.05），建议改用混合Logit或嵌套Logit"), "\n")

# ── 嵌套 Logit 作为 IIA 替代（进阶）──
# 当 IIA 被拒绝时，将备选方案分组为嵌套结构
nl_model <- mlogit(chosen ~ cost + effect + side | 0, data=md,
  nests=list(
    medical = c("A","B"),    # 积极治疗方案
    passive = c("C")         # 不治疗/等待
  ),
  un.nest.el = TRUE  # 估计嵌套相关参数 λ
)
summary(nl_model)
# λ = 1 → 等同于MNL；λ < 1 → 嵌套内备选相关

常见问题与排错指南

整理了 DCE 分析中最频繁出现的报错、异常结果及其解决方案。

📌 mlogit.data() 报错：格式识别失败▾
典型报错：
Error in mlogit.data: 'id' not found
原因：数据中缺少必要的标识变量，或变量名与参数不一致。

解决方案：
1. 确认 id.var 参数指向的列确实存在且为因子类型。
2. 确认 choice 变量为 0/1 整数，不是 TRUE/FALSE 逻辑值。
3. 确认 alt.var 列中每个选择集内的备选方案名称一致（如都是 "A","B" 而非 "A","b"）。
4. 每个受访者的每个任务必须恰好有1行 chosen=1，否则需重新清洗数据。 
table(long %>% group_by(id,task) %>% summarise(n=sum(chosen),.groups="drop") %>% pull(n))
📌 MXL 模型不收敛 / 迭代次数超限▾
典型表现：出现 Warning: did not converge 或迭代次数达到上限。

解决方案：
1. 增加 Halton 序列抽样数：R = 1000（默认500），提高估计精度。
2. 减少随机参数数量，从最重要的1–2个属性开始，逐步增加。
3. 将部分参数从随机改为固定：rpar = c(cost="n")（只有cost随机）。
4. 尝试不同优化算法：method = "bfgs" 或 "nr"。
5. 若价格参数符号异常，改用对数正态：cost = "ln"（但需取负号）。
📌 价格系数为正数（符号方向错误）▾
原因分析：这通常不是代码问题，而是数据或编码问题。

检查步骤：
1. 确认 chosen 变量编码正确（1=选择该方案，0=未选择）。
2. 检查价格变量是否为数值型而非因子型：class(long$cost)。
3. 如果价格以对数形式输入（log(cost)），系数可能为负但绝对值更大。
4. 检查是否存在数据逻辑错误：价格高的方案被系统性地多选（说明价格与质量混淆）。
5. 如果确实正确但符号为正，说明受访者偏好高价格（奢侈品效应），需要在报告中特别讨论。
📌 WTP 计算结果为 NA 或 Inf▾
原因：价格系数估计值接近0，导致 WTP = -β_attr / β_cost 出现除零问题。

解决方案：
1. 检查价格属性的变异是否足够大（水平差距太小则无法准确估计）。
2. 在 Krinsky-Robb Bootstrap 中，过滤掉 β_cost 接近0的抽样： 
wtp_d$effect <- ifelse(abs(draws[,"cost"]) > 0.01,
  -draws[,"effect"]/draws[,"cost"], NA)
3. 考虑改用 logitr 包直接在 WTP 空间估计，避免比值问题： 
library(logitr)
m <- logitr(data=long, outcome="chosen", obsID="task_id",
  pars=c("effect","side"), scalePar="cost")
📌 正交设计行数不足 / oa.design() 无法找到合适设计▾
典型报错：
Error: no orthogonal array found
原因：当属性和水平的组合不对应任何已知正交阵时报错。

解决方案：
1. 调整水平数使其满足正交阵存在条件（如避免混合水平数 2×3×4）。
2. 使用 columns = "order" 参数放宽约束： 
oa.design(nlevels=lvls, columns="order")
3. 改用 D-optimal 设计，它不受正交阵存在性约束。
4. 手动指定正交阵行数（接受部分属性混淆）： 
oa.design(nlevels=lvls, nruns=18)
📌 LCM 各类别系数接近，类别无实质差异▾
原因：样本中实际上偏好较为同质，或模型过拟合。

解决方案：
1. 减少类别数（从 BIC 最优退一步尝试更少类别）。
2. 增加样本量，LCM 至少需要 400 人才能稳定分群。
3. 加入受访者特征变量（年龄、收入等）作为类别成员概率的协变量： 
gmnl(chosen~cost+effect|0|0|0|age+income, model="lc", Q=k)
4. 考虑改用 MXL，其对样本量要求更低且更稳健。
📌 混合 Logit 内存溢出 / 运行极慢▾
原因：当 R（抽样数）和样本量都很大时，模型需要存储和操作大型矩阵。

解决方案：
1. 先用 R=200 快速验证代码，确认无误后再用 R=1000 正式估计。
2. 使用 Halton 序列（halton=NA）代替随机抽样，同样 R 值下精度更高。
3. 考虑改用 apollo 包，它支持并行计算： 
library(apollo); apollo_control(nCores=4)
4. 在 64 位 R 中运行，并增大内存限制：memory.limit(size=16000)（Windows）。
